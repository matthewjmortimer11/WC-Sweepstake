Dethrone UI concepts

Three directions for the play screen after the V3b map. Static mockups only — not wired to game logic. Pick one (or mix) before we implement Phase 20 polish.

A — Kingdom stack

One portrait “kingdom card”: map on top, active location’s action bar as a coloured footer strip, court as slim player stubs along the bottom.

Best for mobile · single scroll

B — Map + court inset

Map stays hero. Court slides in as a role-card gallery on desktop (right rail) or a bottom sheet on phone — tap a player to peek their public role art.

Best for 5–6 players · role art forward

C — Play / Court / Log tabs

Mobile-first tab bar: full-screen map on Play, card-grid Court on second tab, chronicle on third. Desktop keeps side-by-side layout.

Best for small screens · least clutter
